# 心跳唤醒（Heartbeat）

侧栏 → **心跳** → 新建

心跳让 Agent 按固定间隔自我唤醒并执行一段提示词，无需任何用户消息 —— 适合监控、每日汇总、定时主动推送等"周期性检查并执行 X"的工作流。

## 心跳 vs 调度器：何时使用？

| 需求 | 使用 |
|------|-----|
| 每隔 N 秒 / 分钟运行固定提示词 | **心跳** |
| 按 cron 运行（如工作日早上 9 点） | [调度器工具](./tools#调度器scheduler) |
| 一次性提醒 | [调度器工具](./tools#调度器scheduler) |

心跳是挂在 Agent + 目标上的高层循环；调度器工具是 Agent 自己可以调用的低层 cron 原语。

## 配置项

| 字段 | 说明 |
|-------|-------------|
| 名称 | 显示名称 |
| Agent | 每次触发时使用的 Agent |
| 目标 | 具体的渠道用户、Web 会话或工作目录 |
| 间隔 | 触发周期（秒 / 分钟 / 小时） |
| 提示词 | 每次触发时 Agent 收到的提示词模板 |
| 启用 | 切换状态而不删除配置 |

## 示例

- **状态摘要** —— 每 1 小时汇总繁忙的 Lark 群中新消息，并把摘要发到 "summary" 话题
- **看门狗** —— 每 5 分钟通过 Web 工具查询健康检查接口，仅在失败时通知你
- **站会助手** —— 每个工作日早上询问昨天的进展，并写入笔记

## 注意

- 每个心跳目标拥有独立的对话线程（隔离历史）
- 与 [Insight](./insight) 配合，让 Agent 从每次触发中学习
- 禁用心跳可暂停而不丢失配置
