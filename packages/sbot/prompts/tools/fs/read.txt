Reads a text file. Path must be absolute. Output is capped at 50KB.
IMPORTANT: Do NOT use read to search for content inside files. Use grep first to locate exact line numbers, then use offset+limit to read only the relevant section. Reading entire files to find content wastes context and is strongly discouraged.

mode (default "line"):
- "line": offset = 1-indexed start line; limit = max lines (default 2000). Example: offset=50 limit=40 reads lines 50–89.
- "byte": offset = 0-indexed start byte; limit = max bytes (capped at 50KB). Use this for tool-overflow spill files — the truncation marker gives exact byte ranges, plug them in directly.
- "endLine": offset = number of lines back from EOF where the read starts; defaults to limit. Pass just limit=N to read the last N lines.
- "endByte": offset = number of bytes back from EOF; defaults to limit. Pass just limit=N to read the last N bytes.

- Returns "Did you mean?" suggestions when the file is not found.
- Automatically detects and rejects binary files; use read_media_file for binary files.
- Use ls to list directory contents instead of reading a directory path.
