Look up who/what this bot can send to: its channels, group chats (sessions), and users — and their ids. Call this whenever you need to send a message or file but only have a name (e.g. "发给张三 / 通知运维群 / send it to the ops channel") and not yet a numeric id. The id you get here is passed straight into channel_send. Reads from the bot's local config / database — no platform API permission required.

Always returns ALL configured channels nested as <channels><channel>...</channel></channels>. The `type` parameter controls which children are emitted directly under each <channel>:

- type="channel":          no children — channel metadata only.
- type="session":          <session> rows under each <channel>.
- type="user":             <user> rows under each <channel>.
- type="session_and_user": both. Use when you don't yet know whether to address a session or a user.

A <channel> with no matching children is rendered self-closing.

current="1" on a <channel> marks the channel that THIS conversation is happening on. When the user mentions a session/user name without specifying which channel and the name exists on multiple channels, prefer the entry under current="1" — it's almost always what they meant.

Each <session>/<user> exposes only `id` and `name`. Pass that `id` directly to channel_send — it is globally unique across channels. The owning channel (and its caps) is whichever <channel> the row sits under.

caps (on <channel>) is a comma-separated capability set. Tokens:
  text       — channel_send (kind=text) works on sessions of this channel
  file       — channel_send (kind=file) works on sessions of this channel
  text_user  — channel_send (kind=text) works on users of this channel
  file_user  — channel_send (kind=file) works on users of this channel
A token NOT listed means the corresponding call will fail; do not attempt it.
Empty caps usually means the channel is configured but not currently running.

Note: sessions/users only include those the bot has actually interacted with (or that an admin has manually registered).
