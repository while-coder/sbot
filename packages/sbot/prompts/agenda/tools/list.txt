List the user's agenda items, including todos, reminders, scheduled tasks, routines, deadlines, upcoming notifications, and automations.

Use this whenever the user asks what they need to do, what reminders they have, what is scheduled, what is pending, or what is coming up.
Default view should answer "待办/what do I have to do" and include todos, reminders, routines, and upcoming non-automation items. Use view=upcoming when the user specifically asks for scheduled future triggers.

For routines with completionMode=occurrence, the result expands occurrence history grouped by status:
  occurrences: pending=N missed=N done=N cancelled=N
    pending: <ISO times of instances waiting for the user to check in>
    missed:  <ISO times of instances the user did not check in before the next fire>
    done:    <ISO times the user has confirmed>
Times are UTC ISO truncated to minute (YYYY-MM-DDTHH:mm). Each group lists at most 10 most recent times; older entries are summarized as "… N more earlier".

The item header may also carry `allowLateComplete=Y`. Without that flag, missed instances are permanent — the user CANNOT mark a past missed instance done anymore. With the flag (typical for 周报/月报/任务汇报), missed can still be completed via `agenda_complete(at=...)`.

When the user asks "哪几次没做 / 哪天没发 / 我今天漏了几次 / 还有什么待办 / what did I miss" for an occurrence routine,
read the missed and pending lists and answer with the concrete times (e.g. "12:00 和 13:00 没喝水"), not just a count.
Compare against <now> to decide whether a pending instance is "the current one waiting" vs "already overdue and should be confirmed/cancelled".
If the user implies they want to backfill a missed instance ("我现在补交周一的周报"), check allowLateComplete first; only then call agenda_complete with at=that occurrence's scheduledAt.
