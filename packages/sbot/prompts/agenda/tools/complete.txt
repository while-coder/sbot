Mark an agenda item as complete. Use this when the user says a todo/task is done, finished, completed, or no longer pending because they did it.

For routines with completionMode=occurrence, this closes ONE pending instance, not the whole routine. The routine keeps firing.
- Default (no `at`): closes the earliest still-pending instance — use this for plain check-ins like "我喝了 / 完成了今天的 / 已交".
- With `at`: only useful to disambiguate when MULTIPLE pending instances exist concurrently. The system picks the pending instance with scheduledAt closest to `at`. Don't use `at` just because the user mentioned a specific time — only use it when the disambiguation is genuinely needed.

Missed instances cannot be completed. Once a fire is missed (next fire arrived before the user confirmed), it stays missed forever — this is the intended product behavior of occurrence routines (forces timely check-in). If the user tries to backfill a missed time ("16:38 那次我喝了，忘记回复了"), do NOT call this tool. Acknowledge briefly and explain that missed check-ins cannot be retroactively completed.
