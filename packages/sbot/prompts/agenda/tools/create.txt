Create a user agenda item. Use this for todos, reminders, schedules, routines, deadlines, and scheduled automations.

Use this whenever the user asks you to remember something, remind them later, schedule a future notification, create a routine, or run an agent action later.
Prefer structured time fields: at, after, every, or cron. Do not invent a cron expression when after/every/at is enough.

If you are not sure whether the item already exists, call agenda_list first. If this tool says an agenda item already exists, treat that as success and do not create another item.

When creating a routine with completionMode=occurrence, set action=notify_and_record (instead of the default notify):
- The user is expected to reply ("已喝 / 已交 / 完成了") to each fire. Without a recorded fire in the saver, the main agent sees an orphan reply and gets confused.
- notify_and_record writes the fire as an AI-role message into the conversation history so the agent can correctly read short check-in replies.
For non-occurrence reminders (one-shot, simple notifications), keep action=notify (or omit) — recording every fire would inflate the conversation needlessly.
Use action=invoke only when you want the AI to actively respond to the fire (e.g. "每天 8 点帮我总结日志").

Note: occurrence routines have no "late completion" mode. Once a fire is missed (the next fire arrives before the user confirms), that instance stays missed forever and cannot be backfilled. This is the intended product behavior — occurrence routines force timely check-in.
