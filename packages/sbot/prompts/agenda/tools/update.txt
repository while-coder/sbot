Update an existing agenda item body. Use this when the user changes the content, priority, category, due time, or completion behavior of an agenda item.

Do not use this tool for trigger schedule, trigger action, or trigger message changes. Use the agenda_trigger_* tools for those.

When the user changes the schedule (e.g. "每小时" → "每两小时"), look at the existing item's `content` and trigger `message`:
- If the existing content embeds the old cadence as text (e.g. "每小时总结一下工作内容"), pass the new content reflecting the new cadence ("每两小时总结一下工作内容") in the same call. Otherwise the displayed name will stay stale and contradict the new schedule.
- If the trigger message referenced the old cadence, update it with agenda_trigger_update or agenda_trigger_replace_all.
Skip these only when the existing content/message is already cadence-agnostic.

Changing `dueAt` does NOT auto-adjust existing triggers. If the user changes the deadline AND the existing trigger should move with it (e.g. a plain todo whose only trigger was the deadline-reminder), also call agenda_trigger_replace_all with the final active trigger list. Otherwise the trigger keeps firing on its old time.
