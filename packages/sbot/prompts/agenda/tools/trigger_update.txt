Rewrite ONE existing trigger by triggerId. Replaces schedule + action + message; resets fireCount/lastFiredAt.

INPUT SHAPE — flat spec, not nested:
  { triggerId, kind:"absolute", at, action?, message? }
  { triggerId, kind:"interval", every:{amount,unit}, startAt?, count?, action?, message? }
  { triggerId, kind:"cron", expr, startAt?, count?, action?, message? }

The spec must be COMPLETE. To change only action, still send the full spec with the existing kind/schedule (read it from <existing-agenda> XML) plus the new action.

USE WHEN
The user references a specific reminder/schedule already on the item: "move the 9am reminder to 10am", "switch that trigger to invoke mode".

DO NOT USE WHEN
- Adding a new trigger alongside existing ones → agenda_trigger_add.
- Replacing the whole list → agenda_trigger_replace_all.
- Removing → agenda_trigger_remove.

NOTE
fireCount / lastFiredAt are reset. Harmless for unlimited routines; for count-bounded ones progress is lost — rare in practice.
