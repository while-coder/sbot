# 发布说明

本次更新大幅扩展了 sbot 的能力面：新增工作流引擎、3 个聊天渠道，内置内网穿透服务（无需额外反向代理即可对外暴露本地服务器），新增跨平台桌面 / Android Helper 应用（用于需要内嵌浏览器的渠道登录），并显著提升了 MCP 集成的稳定性。

### 工作流引擎

新增工作流系统，可用确定性的控制流编排多步骤的 Agent 任务。工作流执行过程中的错误现在会被妥善捕获并清晰展示，不再导致整个运行崩溃。

### 新增渠道

- **小爱音箱（小米）** —— 通过小米/小爱音箱与 Bot 对话，轮询间隔可配置。
- **钉钉** —— 一等公民的钉钉渠道，适用于企业群聊场景。
- **QQ（OneBot v11）** —— 接入任意兼容 OneBot v11 的 QQ Bot 框架。

### 内置内网穿透

新增内网穿透服务，可在管理后台一键将本地服务器暴露到公网，提供三种方式：Cloudflare Quick（零配置）、Cloudflare Token（自有命名隧道）、Localtunnel。所需的 `cloudflared` 二进制会在首次使用时自动下载。

### 桌面 / Android Helper 应用

新增基于 Tauri 的 Helper 应用，承载需要内嵌浏览器的渠道登录流程，现在同时支持**桌面端和 Android**。内置自动更新、已签名的正式发布版本，首个落地的登录流程为小爱 cookie 登录 —— 在 Helper 窗口完成一次登录，sbot 自动接手会话。

### WebSocket 传输

sbot 在原有 HTTP 接口之外新增 WebSocket 通道，服务端到客户端的推送延迟更低，新版管理后台的实时更新即基于此实现。

### VSCode 会话集成

当 sbot 从 VSCode 启动时，活动会话的工作目录会被自动跟踪和更新 —— 切换文件或工作区会立即反映到 Agent 上下文，无需手动干预。

### 消息压缩

长对话在送入模型前会自动压缩，让长时间会话也能控制好上下文占用，同时保持对话连贯性。

### MCP 连接更稳定

远端 MCP 服务（HTTP / SSE / streamable HTTP）在重启或网络抖动时常会丢失 session，导致工具调用失败。本次更新可识别这类失效连接并自动重连，对 Agent 透明，不再因临时故障打断对话。

### 工具输出处理

超长的工具输出会在送入模型前自动截断，并支持按 Agent 配置上限。`read` 文件工具同步重写，使大文件在长对话中表现更可控；工作路径（workpath）的处理也更稳健。

### 飞书增强

卡片更新失败（超时、消息过期等）现在会被妥善捕获 —— 当配置启用后，回复会自动降级为发送文件，不再丢失响应。

### 调度器

定时任务现在支持禁用而不必删除，更方便地暂停和恢复周期性任务。

### 统一的渠道工具

所有渠道现在向 Agent 暴露同一套工具 —— 列出渠道/会话/用户、发送消息、发送文件 —— 多渠道 Agent 在不同平台间表现一致。

### 文档

文档站完成整体翻新，刷新主题并补齐全量中文翻译，覆盖每个 Guide 页面（Agents、Channels、Tools、MCP、Memory、Skills、Wiki、Heartbeat、Insight、Note、Savers 等）。
