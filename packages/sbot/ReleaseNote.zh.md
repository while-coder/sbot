本次主要更新如下：

### 新增

1. Memory 管理页支持从任务列表直接重试失败的记忆抽取任务。
2. 同步文件索引改为后台排队任务，可以在任务列表里看到处理状态。

### 优化

1. Memory 和 Agenda 的后台队列在启动后恢复更稳定，会自动继续处理上次遗留的任务。
2. Memory 任务日志会显示记忆配置名称、模型信息和更清晰的失败原因，排查问题更容易。
3. 结构化输出对 OpenAI 兼容接口、Anthropic、Gemini、Ollama 等模型更兼容，会在可用方法之间自动回退。

### 修复

1. 修复同步索引、整理记忆和抽取记忆可能同时写入文件或数据库，导致任务状态不稳定的问题。
2. 修复失败的记忆抽取任务缺少后台重试入口的问题。
