本次主要更新如下：

### 新增

1. 新增 `sbot update` 命令，可更新到最新版本；如果服务正在运行，会先停止服务再升级。
2. 聊天内新增更多内置命令，包括 `/status`、`/workpath`、`/agent`、`/saver`、`/note`、`/wiki`、`/memory`、`/agenda`、`/autoapprove`、`/tokens`、`/version`、`/history`。
3. Agenda 触发器新增更完整的管理能力：手动触发、删除触发器、为单个触发器选择投递会话，以及逐次打卡模式。
4. 钉钉渠道新增文件发送，并支持从已保存的会话信息恢复投递目标，重启后也能继续主动发送。
5. ReAct 多 Agent 新增子任务追踪：可查看任务列表、续接任务状态、按需注入主对话上下文，并限制过深嵌套。
6. Agent 管理新增引用统计、未使用提示、「使用全部」模式下排除指定技能，以及自动生成子 Agent 能力描述。

### 优化

1. 优化渠道发送 / 列表工具提示词，让 AI 能先按名称查找联系人或群聊，再发送消息或文件。
2. 优化 Lark 消息历史读取，返回发送者名称，并改进分页说明。
3. 优化生成文件的交付链接，保存到工作目录多级子目录的文件也能正确生成下载地址。
4. 简化 Agenda 概念，将原来的分类 / 完成方式组合改为更清晰的「逐次打卡」开关。
5. 精简发布与辅助工具链，移除旧 helper 包流程，并将小爱凭据获取说明切换到 sbox。

### 修复

1. 修复 Agenda 触发器启动和重载时的竞态问题，避免调度丢失或重复触发。
2. 修复多触发器事项过早完成的问题，现在会等所有启用触发器都耗尽后才自动完成。
3. 修复 Agenda 投递目标和渠道会话元数据保存问题，提升重启后提醒与主动消息投递的稳定性。
4. 修复 Windows 下执行命令或清理进程时可能弹出控制台窗口的问题。
5. 修复内置命令结果在聊天中被当成普通 AI 回复的问题。
6. 修复嵌入式客户端在无法直接打开浏览器链接或 WebSocket 时的文件下载与终端连接问题。
