# Code Review Skill

## 适用场景

当用户请求代码审查、代码分析、检查代码质量、找 bug、安全审计时使用此 skill。

## 审查维度

### 1. 代码质量
- 命名规范是否清晰易懂
- 函数/方法职责是否单一
- 代码重复度（DRY 原则）
- 注释是否充分

### 2. 安全性
- 输入验证与边界检查
- SQL 注入、XSS 等常见漏洞
- 敏感信息泄露风险
- 权限控制

### 3. 性能
- 算法复杂度分析
- 不必要的循环或重复计算
- 内存使用
- 异步/并发处理是否合理

### 4. 最佳实践
- 错误处理是否完善
- 依赖版本是否安全
- 代码可测试性

## 输出格式

使用以下格式输出审查结果：

```
## 代码审查报告

### 总体评分: X/10

### 发现的问题

#### 严重 🔴
- [问题描述] (行号/位置)
  建议: [修复建议]

#### 警告 🟡
- [问题描述]
  建议: [修复建议]

#### 建议 🟢
- [优化建议]

### 优点
- [代码亮点]

### 总结
[整体评价和改进方向]
```
